PhoneBook_OnClass2(정렬 추가), 예외 처리기, 예외 발생, 사용자정의 예외, 싱글톤 방식

정렬의 기준을 주려면 
컴패어러블은 클래스 안에 한가지 기준을 정해줄때는 사용하고
컴패어러는 아예 비교만을 위한 클래스를 만들어서 쓰고싶을때 사용

예외처리기 : 코딩을 아무리 잘해도 꼭 try~catch를 써야할때
		1. 네트워크 연결되는 부분 (통신, DB)
		2. 파일 처리 (파일을 읽고쓰는 부분)
		3. 사용자에게 입력을 받을때 (사용자가 뭘 입력할지 모릅니다)

예외 발생 : throw : 1. 시스템이 발생시킨 예외가 발생했을때 이 메서드를 호출한 함수에게 던지는 것 => 이 메서드를 부른애가 캐치문이 있으면 예외가 처리 되었다 보고 
										       아닐시에 [ 처리되지 않은 예외 ]로 보고 비정상 종료 
		2. 예외를 의도적으로 발생시켜 캐치로 던지는것

	  throw 와 throw ex의 차이 : throw ex는 Exception타입의 ex이기 ex로 받은 메서드만 보여줌
				throw 는 바로 던졌으니 call stack을하면 자신을 호출한 함수로 던지는것
			            => call stack의 차이
	  throw new ApplicationException => 비지니스 로직 처리할때많이씀 하지만 throw new Exception을 더 많이쓰는편

사용자정의 예외 : Exception은 타입이다. 그래서 별도로 클래스로 만들어 사용할수 있는데 
		사용자 정의 에외는 [ Exception클래스를 부모 ]로 두는 것을 권장.
	
오늘 과제 : 폰북에서 사용자가 잘못 입력했을때 예외처리 추가 , 싱글톤으로 교체
	폰북의 폰북매니저의 뉴를 1번만 하게 싱글톤적용
